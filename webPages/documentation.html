Four Categories

Overall Theme/Impact

The overall theme of the game is turn based strategy. It is basic in style but works well with the mechanics of turn based. The player is meant to think about their position before they decide to shoot becuase if they don't they will be leaving themselves open to an attack from the enemy without anything they can do. For engagement, I believe the to be very engaging. It rewquires direct input from the user as well as some variables the user is allowed to change to make the game harder or easier. The functionality is well beyond what we have done in class and shows where we can take what we have learned and use it within a game. This project is definitely approaching portfolio quality. There are a few parts that I would like to add, such as game assets but other than that I think it's ready.

User Experience

As this app is a game, there are obvious things that need to be mentioned. Users could just jump in and try to learn as they go and they would do okay after a few failures. However, we supplied good information on how things work and what the objective is. The user is able to control not only the variables that make the game harder or easier but they also control the character themselves. It is up to them to make the choices where to stand and when to shoot. The design is basic but I think it works well for what it is. Ideally we would like to add game assets as we have already mentioned.

Media

All css and html has been tested and passes validation. It includes the main required semantics as well as an embedded font that adds to our overall theme. Unfortunatelty we do not have any images to spice up the vibes of the game so there is no optimization needed for them. All canvas drawing uses save() and restore() as required and within the game we use rectangles and arcs (for now).

Code

Now the coding is what we put the most time into. There was a lot of backend stuff that we needed to get just right so it worked properly. All coding standards are upheld and the use of ES6 modules helped clean it up greatly. As we started this project we had the basic idea of having a turn based game so when we started coding we split up the classes into their own js files and so we could each work on specific pieces of the game. This was done by using ES6 Modules.

Features

What went right:

  1. We wanted to create a turn based game which only allowed the player to move during their turn. We created this by having the player's turn end when they shot a bullet or they clicked end turn button. Also we had to create a time function that would count the duration of the random walker's turn in seconds. This allowed for distinct turn cycles between the player and enemies.
  2. We wanted to game world to be bigger than the canvas. We did this by having the player stay in the center of the screen and everything around them move instead. This allowed for a feel that the player was still moving even though they were not. Lastly, this allowed us to extend beyond the canvas and create move objects outside this zone that the player could still interact with.

What went wrong:

  1. The early developmental process for this project did not start off on a great footing. We did not know exactly what we wanted to do for this project and as a group we were not always on the right page about what the game should be. After we had to rethink the game we had similar ideas and finally construction of the game could move forward.
  2. We had some issues with the collisions with everything after we tried to create a one function method for collision with the walls. To do this we had to redo how the walkers were being drawn so they also fit the confinds of this method. This created errors with collision with the player when checking this on the enemies turn.

Features we wanted to add:

  1. We wanted to create a way to collect powerups on the map that let you upgrade you gun for a calculated amount of shots. We wanted to have certain powerups such as shooting 3 bullets in a cone or being able to shoot multiple times in a turn. These would have allowed to add more interactivity to the game for players as they picked up powerups.
  2. We also wanted to create phylotaxis whenever an enemy died. They would start out at the origin of where the bullet hit the enemy. This would have added some color and created the game a bit more interesting.
  3. We wanted to add assets to the game as right now we only have square blocks to represent our game pieces. This would have added more flavor to our game and would have created a better theme as everything would represent something instead of a square.

Non-Course Resources

The other code that we used that we got help on was a section on mouse clicking but, we took that from one of the canvas homework assignments.
